# [PRD] 마이링크 (My-Link) 서비스 기능 정의서

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (My-Link)
- **목적**: 개발자와 크리에이터를 위한 심플하고 감각적인 링크 통합 관리 서비스. 흩어져 있는 포트폴리오, 블로그, SNS 링크를 하나의 페이지로 요약하여 제공합니다.
- **비전**: "나를 표현하는 가장 쉬운 한 페이지"

## 2. 타겟 사용자
- **개발자**: 깃허브, 기술 블로그, 포트폴리오 사이트를 깔끔하게 공유하고 싶은 사용자.
- **크리에이터**: 유튜브, 인스타그램, 작업물 링크를 하나의 URL로 브랜딩하고 싶은 사용자.

## 3. 핵심 기능 목록

### 3.1 필수 기능 (Phase 1)
| 구분 | 기능명 | 상세 내용 |
| :--- | :--- | :--- |
| **인증** | 구글 소셜 로그인 | Firebase Auth를 통한 간편 가입 및 로그인 |
| **프로필** | 개인 페이지 커스텀 | 프로필 이미지, 닉네임, 한 줄 소개 설정 |
| **링크 관리** | 링크 추가/수정/삭제 | 무제한(또는 제한된) 링크 생성 및 활성화 여부 토글 |
| **UI/UX** | shadcn/ui 기반 테마 | 기본적인 스타일 프리셋 제공 및 반응형 레이아웃 |
| **공유** | 고유 URL 제공 | `domain.com/[user-id]` 형태의 고유 접근 경로 제공 |

### 3.2 선택 및 확장 기능 (Phase 2)
| 구분 | 기능명 | 상세 내용 |
| :--- | :--- | :--- |
| **통계** | 방문자 분석 | 일별 방문자 수, 링크 클릭 수 추적 |
| **관리** | 순서 변경 (D&D) | 드래그 앤 드롭을 통한 링크 노출 순서 관리 |
| **디자인** | 상세 커스텀 | 폰트, 배경 이미지, 버튼 모양 등 세부 디자인 시스템 적용 |
| **SEO** | 검색 엔진 최적화 | 개별 페이지의 메타 태그(Title, Description) 설정 |

## 4. 상세 기능 설명

### 4.1 인증 및 보안
- **방식**: Firebase SDK를 이용한 Google OAuth 2.0 적용.
- **데이터 관리**: 사용자 고유 UID를 기반으로 데이터 접근 권한(Security Rules) 제어.

### 4.2 링크 관리 시스템 (Admin)
- **링크 구성 요소**: 타이틀, URL, 아이콘(선택), 노출 여부(Toggle).
- **실시간 미리보기**: 어드민 페이지 내에서 수정 사항을 실시간으로 확인 가능한 모바일 프리뷰 뷰어 제공.

### 4.3 프로필 및 디자인 (shadcn/ui 기반)
- **컴포넌트 구조**: Card, Button, Avatar, Input 등 shadcn/ui의 기본 요소를 활용하여 일관된 디자인 시스템 구축.
- **테마 시스템**: 다크 모드/라이트 모드 지원 및 브랜드 컬러(Primary Color) 적용 기능.

### 4.4 배포 및 라우팅
- **Next.js App Router**: 다이나믹 라우팅(`app/[slug]/page.tsx`)을 통한 사용자 페이지 생성.
- **최적화**: 이미지 최적화(next/image) 및 빠른 페이지 로딩 속도 유지.

## 5. 기술 스택
- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS, shadcn/ui
- **Backend/Auth**: Firebase (Authentication, Firestore)
- **Deployment**: Vercel (권장)
